# archaeo-pro — Design Context

This file holds the **immutable design context** for archaeo-pro. Anything that
changes the way the interface looks or feels should be checked against the
sections below before any code is written.

## Users

**Primary**: Italian *liberi professionisti archeologi* — solo archaeologists
employed by construction sites, infrastructure projects, and Sovrintendenze
to conduct *sorveglianze archeologiche*. They are the legal direttore tecnico
of the surveillance, working alone or with a small team.

They use archaeo-pro in two distinct contexts, both of which the interface
must serve well:

- **In the field** — on a phone or tablet, outdoors, often in direct sun, in
  a trench or on a dusty construction site. Hands are frequently dirty,
  muddy, or gloved. Mobile data is patchy; sometimes it's gone entirely. The
  archaeologist needs to drop a pin, photograph a finding, jot a stratigraphic
  note, and move on without losing data.
- **In the office** — on a desktop browser the same evening or the next
  morning, finalizing the report. Editing free-form sections, attaching
  documentation, generating the DOCX/PDF that gets sent (or printed-and-
  signed) to the SABAP. The work has weight: this document goes on file at a
  ministry, can be cited, and is a legal artifact.

**The job**: produce a Sovrintendenza-grade document quickly and credibly,
while preserving structured data so it can later go to OpenHistoryMap.

**The emotional register**: this is administrative and legal work with
cultural-heritage stakes. The interface needs to project competence — both
to the archaeologist (so they trust it on a real job) and through the
output (because what comes out gets pasted into official communications).

## Brand Personality

Three words: **field-craft, scholarly, Italian.**

- **Field-craft**: archaeo-pro should look like a tool *made for the work*,
  not for a VC pitch. A bit of physicality. The reference is analog field
  instruments — theodolites, planimeters, ranging rods, Munsell soil charts —
  without being kitschy-retro about it. Working surfaces, not glass cards.
- **Scholarly**: classical typographic proportion. Hierarchy comes from
  size, weight, and space — never from rainbow color or decorative chrome.
  Body type is comfortable for long-form note-writing. Headings have gravity.
  This is closer to Italian academic publishing (Marsilio, Quodlibet, Donzelli)
  than to a SaaS landing page.
- **Italian**: borrowed from Italian editorial and product-design tradition —
  Olivetti graphics, Pirelli calendars, Bompiani book covers, the spine of a
  Garzanti dictionary. **Not** from tourist clichés (no terracotta-and-
  cypress sunsets, no stylized Colosseum). Warmth via tinted neutrals; weight
  via real typography.

## Aesthetic Direction

**Spirit** of Felt, Mapbox Studio, and Linear — clean, sharp, restrained,
unmistakably a tool — translated into archaeological/scholarly language.
Linear's typographic rigor and information density. Felt's confident map UI
and willingness to reduce. Mapbox Studio's quiet utilitarianism.

**Theme**: both, user-toggleable. **Default light**, because (a) field use
happens in daylight and high-contrast paper-white reads better in the sun,
and (b) the deliverable is a printed/signed document, so a paper-shaped
interface aligns with the user's mental model. Dark mode for evening
office finalization.

**Palette**: warm-neutral surfaces tinted toward parchment / terra / ochre.
A single confident accent (terra / iron-oxide / Pompeian-red family)
reserved for action, not decoration. Pure cool grays are wrong here; pure
black/white are wrong everywhere.

**Anti-references** (do NOT do):
- Tourist Italy: terracotta + cypress + sunset gradients, Tuscany filters.
- Silicon-Valley SaaS: rainbow brand color, gradient text, AI-cyan-on-dark.
- Academic-by-default: Times Roman everywhere, beige institutional pages.
- Glassmorphism / blur cards / neon glow / icon-rounded-square-above-every-
  heading patterns.

## Design Principles

These are the test every design decision passes before shipping.

1. **Tools, not trinkets.** Every visual element justifies its weight by
   helping a working archaeologist on a dig site. If a flourish doesn't
   help in a trench, cut it.
2. **Scholarly proportion over decorative noise.** Hierarchy is built from
   type size, weight, and white space. Color and graphics are sparing. A
   spread should have the calm of a well-set page, not the busyness of an
   ad.
3. **Restrained palette tinted to earth.** Warm neutrals (parchment,
   sand, slate) plus a single accent. No rainbow, no gradients-as-emphasis,
   no per-section colors.
4. **Field-readable by default.** Body text ≥ 16 px (18 px preferred for
   surveillance content). Touch targets ≥ 44 px. Contrast meets WCAG AA on
   the strictest reading because the field also tests it. Form inputs are
   not allowed to look fragile.
5. **Italian without postcards.** Reach for editorial-typography traditions
   (proper Italian quotation marks « », correct ligatures, dignified
   headings) rather than tourist iconography.

## Specific guidance for Claude

- **Typography**: do NOT default to Inter, DM Sans, Plus Jakarta, IBM Plex,
  Fraunces, Newsreader, or any of the impeccable reflex-reject list. Look
  further. A serif-display + grotesque-text pairing or a single high-quality
  text family with strong weight contrast both work for this brand.
- **Italian copy first.** UI strings are Italian. Technical archaeology terms
  (US, SABAP, CTR, vincolo, sorveglianza) are preserved verbatim — never
  Englished. English is a fallback if needed.
- **Map is the centerpiece** in field mode. Treat it like Felt does: confident,
  full-bleed, with chrome that gets out of the way.
- **The DOCX template** is a related but separate design surface — its
  typographic choices should feel like a sibling of the web UI, not a
  contradiction.
- **Keep `--hex-named` tokens out**. Use OKLCH with semantic names
  (`--surface`, `--surface-warm`, `--text`, `--text-soft`, `--accent`,
  `--accent-quiet`, `--rule`).

## What this context does NOT mandate

- Specific font names — pick fonts that fit the brief, don't lock them in.
- Specific accent hue — pick from the terra/iron-oxide family, but the exact
  OKLCH triple is up to the implementing pass.
- Page layouts — design principles guide them; they're not pre-decided.
